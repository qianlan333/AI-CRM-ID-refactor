## 引用核验

- Required candidate grep shows deleted modules are no longer present in `wecom_ability_service/http/__init__.py`.
- `aicrm_next/main.py` does not import `production_compat_router`、`production_compat_wildcard_router`、`forward_to_legacy_flask`、`legacy_flask_facade` or `X-AICRM-Compatibility-Facade` response path.
- Current route resolution counters remain zero for production compatibility, legacy fallback, unknown owner, undocumented routes, and deleted-but-registered routes.

## 后续删除条件

1. 将 legacy explicit fallback app 的 historical HTTP registration tests 改为 domain-service level tests 或 Next API tests。
2. 将 `tests/test_automation_conversion_v1.py` 中对 `wecom_ability_service.http.automation_conversion` 的 monkeypatch 迁到 `aicrm_next.automation_engine` safe-mode boundary。
3. 将 `tests/integration/test_pg_compat_smoke.py` 中 cloud orchestrator monkeypatch 迁到 `aicrm_next.cloud_orchestrator` command boundary。
4. 将 `aicrm_next.integration_gateway.legacy_sidebar_read_facade` 的 customer automation helper 迁到 Next read model，再删除 `customer_automation.py`。
