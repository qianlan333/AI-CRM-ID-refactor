# Server Readonly Evidence Runbook

## Business Impact

This evidence pack helps verify whether the live server is actually serving
AI-CRM Next health, admin, customer, questionnaire, sidebar, route owner, and
production data surfaces. It prevents treating local checker output as proof
that production is ready.

## Server Command

Run on the production server:

```bash
cd /home/ubuntu/极简\ crm
set -a
source /home/ubuntu/.openclaw-wecom-pg.env
set +a
source /home/ubuntu/venvs/openclaw/bin/activate
python3 tools/collect_server_readonly_evidence.py \
  --base-url http://127.0.0.1:5001 \
  --output-md /tmp/server_readonly_evidence.md \
  --output-json /tmp/server_readonly_evidence.json
```

## What It Collects

- `/health`
- `/api/system/health`
- `/admin`
- `/admin/customers`
- `/admin/questionnaires`
- `/admin/automation-conversion`
- `/admin/jobs`
- `/sidebar/bind-mobile`
- `/api/customers?limit=1`
- `/api/admin/questionnaires?limit=1`
- route owner headers
- `X-AICRM-App`
- `X-AICRM-Release-SHA`
- fixture/local_contract/demo marker scan
- `production_data_ready`
- `database_mode`
- safety flags for timers, external calls, and legacy fallback removal

## Evidence Classes

- Local checker evidence: generated on a developer machine or local probe.
- Server readonly evidence: generated by this script against
  `http://127.0.0.1:5001` on the server.
- Production canary evidence: a separate approved server-side canary process
  with SHA/header proof and explicit approval language.

This script only creates server readonly evidence. It is not automatic
production cutover approval.

## Safety Boundaries

- Do not modify nginx/systemd/deploy config.
- Do not enable timers.
- Do not run write routes.
- Do not enable real WeCom, Payment, OAuth, OpenClaw, or MCP external calls.
- Do not remove legacy fallback.
- The output files must not be described as production canary approval.

## Interpreting Output

- `ok=false` means one or more readonly probes found blockers.
- `database_mode=postgres` with fixture/local_contract/demo markers is a
  blocker.
- Missing `X-AICRM-Route-Owner`, `X-AICRM-App`, or `X-AICRM-Release-SHA` is a
  blocker because route ownership and release identity are unclear.
- `safe_to_enable_timers`, `safe_to_enable_real_external_calls`, and
  `safe_to_remove_legacy_fallback` remain false.
