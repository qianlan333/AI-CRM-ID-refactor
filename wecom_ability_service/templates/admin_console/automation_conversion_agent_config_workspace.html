上下文注入规则

这里不再单独勾选 4 类信息来源。你只需要在角色提示词或任务提示词里写下对应占位符；运行时系统会自动识别被引用的项，并且每类信息只整理一次放进上下文，不会因为重复写占位符而重复膨胀。

  • 写了 {% raw %}{{问卷信息}}{% endraw %}，才会注入问卷题目与答案摘要。
  • 写了 {% raw %}{{最近20条聊天信息}}{% endraw %}，才会注入最近 20 条聊天信息。
  • 写了 {% raw %}{{用户标签}}{% endraw %}，才会注入当前用户标签。
  • 写了 {% raw %}{{激活信息}}{% endraw %}，才会注入激活状态、当前池子和当前大人群。
变量插入区

点一下就能把占位符插入到当前聚焦的提示词编辑框。没有聚焦时默认插入到任务提示词。占位符在提示词里重复出现也不会重复展开原始内容，只会引用同一份上下文块。

系统内部固定只生成一条话术，保存和发布时会自动按 `draft_reply` 收口，不再让你配置输出协议 JSON。
